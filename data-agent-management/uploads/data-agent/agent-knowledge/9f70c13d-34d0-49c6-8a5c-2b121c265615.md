# 供排水管网元数据与查询规则

## 1. 管网识别入口
- 先查 M_DBMETA，优先使用 describe 识别用户提到的供水管网或污水管网。
- 识别出 describe 后，再取对应 layername。
- 当前真实管网前缀只有两套：HZGS=供水管网，WS=污水管网。

## 2. 真实表族定位
- 识别出 layername 后，再定位 {prefix}_lin、{prefix}_nod、{prefix}_M_MT、{prefix}_M_MT_FLD。
- HZGS 对应供水管网表族。
- WS 对应污水管网表族。

## 3. 设备类型识别
- {prefix}_M_MT 是设备类型字典。
- dno 是设备类型编号。
- dname 是设备名称。
- dalias 是更适合自然语言匹配的中文别名。
- 用户提到阀门、水表、消火栓、设施井等设备时，应先通过 dname 或 dalias 找到 dno。

## 4. 字段映射规则
- {prefix}_M_MT_FLD 是字段字典。
- name 是真实数据库字段名。
- alias 是中文显示名。
- prop 是字段特性码。
- 遇到管长、管材、管径、埋深、高程、井类型等业务词时，必须先根据 prop 找真实字段 name，再生成 SQL。

## 5. 查询落点规则
- 设备统计默认优先走 {prefix}_nod。
- 管长、管材、管径、起终点等线段统计默认优先走 {prefix}_lin。
- 不允许直接猜字段名。
- 不允许把设备统计错误落到 lin。
- 不允许把长度统计错误落到 nod。

## 6. 当前已确认的关键事实
- HZGS_M_MT 有 51 个设备类型。
- WS_M_MT 有 2 个设备类型：管段、设施井。
- HZGS_M_MT_FLD 中已确认有 PIPELENGTH、PIPEMATERIAL、PIPEDIAMETER。
- WS_M_MT_FLD 中已确认有 PIPELENGTH、PIPEMATERIAL、PIPEDIAMETER 对应的真实字段。
- WS_* 结构存在，但当前表中基本无数据，因此 SQL 路径要正确，不要求结果一定非空。
